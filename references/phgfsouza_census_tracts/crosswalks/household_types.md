# Household Types Crosswalk

This folder contains a crosswalk for harmonizing household and dwelling species between the 2010 and 2022 Brazilian Censuses. The source term is `domicílio`; the harmonized English labels use "dwellings" because the crosswalk includes both occupied and non-occupied categories. 

This document was generated by Pedro H. G. Ferreira with the help of GPT-5.5 on 2026-04-25.

## Files

- `household_types.csv`: machine-readable crosswalk. Use this file in scripts.
- `household_types.md`: documentation for the crosswalk, including assumptions and caveats.

## Recommended Harmonization

| Harmonized code | Harmonized label | 2010 source categories | 2022 source categories |
|---|---|---|---|
| `pp_occ` | Private permanent occupied dwellings | Domicílio Particular Permanente Ocupado; Domicílio Particular Permanente Fechado when using final imputed aggregates | Domicílio Particular Permanente Ocupado (DPPO) |
| `pi_occ` | Private improvised occupied dwellings | Domicílio Particular Improvisado Ocupado | Domicílio Particular Improvisado Ocupado (DPIO) |
| `c_occ` | Collective occupied dwellings | Domicílio Coletivo com Morador | Domicílio Coletivo Com Morador (DCCM) |
| `pp_ocuse` | Private permanent occasional-use dwellings | Domicílio Particular Permanente de Uso Ocasional | Domicílio Particular Permanente de Uso Ocasional (DPPUO) |
| `pp_vac` | Private permanent vacant dwellings | Domicílio Particular Permanente Vago | Domicílio Particular Permanente Vago (DPPV) |
| `c_nonocc` | Collective non-occupied dwellings | Domicílio Coletivo sem Morador | Domicílio Coletivo Sem Morador (DCSM) |

**Notes:**

- Use `pp_occ`, `pi_occ`, and `c_occ` for the comparable occupied resident-household universe.
- Keep `pp_ocuse`, `pp_vac`, and `c_nonocc` for dwelling-stock, coverage, or field-operation checks, but exclude them from resident-household comparisons.
- In 2010, `Domicílio Particular Permanente Fechado` means the dwelling had residents on the reference date, but the interview was not completed because the residents were absent. The 2010 dictionary describes an imputation procedure for these dwellings and says the imputed residents and dwelling characteristics were aggregated into the Census results.
- For final published 2010 aggregate results, map `Domicílio Particular Permanente Fechado` to `pp_occ`. For raw collection-status data, preserve the closed status separately, for example as `source_occupancy_status == "occ_no_interview_imputed"`, and decide whether to include it based on the analysis.

## Accounting Identities

Useful identities for analysis:

| Identity | Meaning |
|---|---|
| `total_occ = pp_occ + pi_occ + c_occ` | All occupied dwellings in the comparable resident-household universe. |
| `p_occ = pp_occ + pi_occ` | All occupied private dwellings. This corresponds to 2022 `DPO` (`Domicílio Particular Ocupado`). |
| `pp_total = pp_occ + pp_ocuse + pp_vac` | All permanent private dwellings in the harmonized stock categories. |
| `p_total = pp_total + pi_occ` | All private dwellings covered by this crosswalk. |
| `c_total = c_occ + c_nonocc` | All collective dwellings covered by this crosswalk. |
| `total_units = pp_occ + pi_occ + c_occ + pp_ocuse + pp_vac + c_nonocc` | All dwelling units covered by the harmonized categories. |

For 2022, the basic-file identities map directly to the source acronyms:

| 2022 source identity | Harmonized identity |
|---|---|
| `DPO = DPPO + DPIO` | `p_occ = pp_occ + pi_occ` |
| `Total de Domicílios Particulares = DPPO + DPPV + DPPUO + DPIO` | `p_total = pp_occ + pp_vac + pp_ocuse + pi_occ` |
| `Total de Domicílios Coletivos = DCCM + DCSM` | `c_total = c_occ + c_nonocc` |
| `Total de Domicílios = DPPO + DPPV + DPPUO + DPIO + DCCM + DCSM` | `total_units = pp_occ + pp_vac + pp_ocuse + pi_occ + c_occ + c_nonocc` |

## Field Definitions

| Column | Meaning |
|---|---|
| `census_year` | Census year, `2010` or `2022`. |
| `source_type_code` | Source code or normalized source key. For 2022, official acronyms are used where available. For 2010, normalized keys are used because the documentation does not expose equivalent acronyms for all species. |
| `source_type_label` | Original source category label. |
| `source_is_original_code` | `yes` when `source_type_code` is an official source acronym; `no` when it is a normalized key created for this crosswalk. |
| `source_occupancy_status` | Broad occupancy or response-status interpretation of the source category. Values are `occ`, `occ_no_interview_imputed`, `ocuse`, `vac`, and `nonocc`. |
| `harmonized_type_code` | Concise harmonized category code for analysis. |
| `harmonized_type_label` | English label for the harmonized category. |
| `notes` | Mapping caveats and recommended handling. |
| `source_reference` | Documentation locations used to justify the mapping. |
